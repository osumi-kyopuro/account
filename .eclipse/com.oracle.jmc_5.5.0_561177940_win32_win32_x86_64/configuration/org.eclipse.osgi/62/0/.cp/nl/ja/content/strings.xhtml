JMXコンソールは、実行中のJVMインスタンスのモニタリングと管理のためのツールです。このツールによって、メモリーとCPU使用状況、ガベージ・コレクション、スレッド・アクティビティなどのライブ・データが提供されます。また、JVMおよびJavaアプリケーションのMBeanのモニターおよび管理に使用できる完全な機能を持つJMX MBeanブラウザも含まれています。
Java Flight Recorderは、HotSpot JVMと実行中のアプリケーションに関する詳細な記録を生成します。フライト・レコーダ・ツールを使用して、記録されたプロファイルをオフラインで分析できます。記録されたデータには、メソッドとロックのプロファイリング、ガベージ・コレクションの統計、最適化の決定、イベントの待機時間などが含まれています。
Oracle Java Mission Controlは、Javaアプリケーションの管理、モニタリング、プロファイリングおよびトラブルシューティングを行うためのツール・スイートです。初回インストール時に、Java Mission ControlはJMXコンソールとJava Flight Recorderで構成されます。Mission Controlから他のプラグインを簡単にインストールできます。
Mission Controlの機能を拡張したプラグインが使用可能です。これらはすべてMission Controlからインストールできます。「ヘルプ」→新規ソフトウェアのインストール・メニューに移動し、インストールするプラグインを選択します。使用可能なプラグインの例をいくつか示します。
Java Mission Controlクライアントは、JVMに接続して、Java Mission Controlに含まれるツールを開始できるアプリケーションです。
JavaアプリケーションとJVMの実行時のモニターおよび管理を行います。
JVMの実行時の動作を記録し、Java Flight Recorderツールを使用してオフラインで記録の分析を行います。
Mission Controlの機能を拡張するために様々な特殊プラグインをダウンロードしてインストールします。

メモリー使用状況、ガベージ・コレクション、頻繁に使用されるメソッドとクラス、スレッド、Javaロックなどに関する詳細な情報を表示します。
イベントの分析
「イベント」タブを使用して、Javaアプリケーションの待機時間を特定および分析します。
JOverflow
ヒープのコンテンツを分析して、メモリーが不必要に浪費されているかどうかを確認します。分析では、重複する文字列、疎/空のコレクションなどの様々なアンチパターンを使用してメモリー浪費を探します。
Weblogic Server Tab Pack for JFR
Weblogicのカスタム設計タブを使用してWeblogic Serverからフライト記録のコンテンツをより効率的に分析します。新規タブ・グループがフライト・レコーダに追加され、これによりデータベース問合せ、EJBコール、サーブレット呼出しなどを参照できます。
DTrace
Mission ControlでDTrace記録を分析します。Dscriptに新しい言語拡張を追加して、記録するデータを自分で記述できます。Java Flight Recorderで実行するのと同様の形式で分析できます。
